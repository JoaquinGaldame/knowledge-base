# Architecture

Esta seccion organiza patrones, estilos y decisiones arquitectonicas para sistemas de software.

## Secciones principales

- [ADR](./adr/) - Registro y analisis de decisiones arquitectonicas.
- [Clean Architecture](./clean-architecture/) - Separacion de capas, dependencias y casos de uso.
- [Event-Driven Architecture](./event-driven/) - Integracion por eventos, asincronia y desacople.
- [Hexagonal Architecture](./hexagonal/) - Puertos, adaptadores y aislamiento del dominio.
- [Layered Architecture](./layered/) - Estructuras por capas para aplicaciones empresariales.
- [Microservices](./microservices/) - Servicios independientes, limites y tradeoffs operativos.
- [Monoliths](./monoliths/) - Monolitos modulares, simplicidad inicial y evolucion.
- [Onion Architecture](./onion/) - Centro de dominio y dependencias hacia adentro.

## Enfoque de la biblioteca

El objetivo de esta area es comparar estilos de arquitectura, documentar tradeoffs y construir criterio para elegir estructuras sostenibles segun contexto.
