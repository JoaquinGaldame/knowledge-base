# Structural Design Patterns

## Introducción

Los patrones estructurales se enfocan en cómo organizar y componer clases y objetos para construir sistemas más flexibles y mantenibles.

Su objetivo es simplificar las relaciones entre componentes y reducir el acoplamiento.

## Patrones incluidos

### Adapter

Permite que interfaces incompatibles trabajen juntas.

- [Ir a Adapter](./adapter.md)

### Facade

Proporciona una interfaz simplificada sobre un subsistema complejo.

- [Ir a Facade](./facade.md)

### Decorator

Permite agregar funcionalidades dinámicamente sin modificar la implementación original.

- [Ir a Decorator](./decorator.md)

### Composite

Permite tratar objetos individuales y grupos de objetos de forma uniforme.

- [Ir a Composite](./composite.md)

### Flyweight

Reduce el consumo de memoria compartiendo datos comunes entre múltiples objetos.

- [Ir a Flyweight](./flyweight.md)

### Proxy

Controla el acceso a un objeto mediante un intermediario.

- [Ir a Proxy](./proxy.md)

## Cuándo estudiar estos patrones

* Cuando se necesita integrar sistemas incompatibles.
* Cuando existen subsistemas complejos.
* Cuando se busca reutilización mediante composición.
* Cuando se necesita controlar acceso o consumo de recursos.
