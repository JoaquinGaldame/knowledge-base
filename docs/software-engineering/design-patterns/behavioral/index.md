# Behavioral Design Patterns

## Introducción

Los patrones de comportamiento se enfocan en la comunicación y colaboración entre objetos.

Su objetivo es distribuir responsabilidades de manera adecuada y reducir el acoplamiento en la lógica de negocio.

## Patrones incluidos

### Strategy

Permite intercambiar algoritmos dinámicamente.

- [Ir a Strategy](./strategy.md)

### Observer

Permite notificar automáticamente cambios de estado a múltiples observadores.

- [Ir a Observer](./observer.md)

### Command

Encapsula operaciones como objetos.

- [Ir a Command](./command.md)

### Chain of Responsibility

Permite procesar solicitudes mediante una cadena de manejadores.

- [Ir a Chain of Responsibility](./chain-of-responsibility.md)

### Iterator

Permite recorrer colecciones sin exponer detalles internos.

- [Ir a Iterator](./iterator.md)

### Mediator

Centraliza la comunicación entre múltiples componentes.

- [Ir a Mediator](./mediator.md)

### Memento

Permite capturar y restaurar estados previos.

- [Ir a Memento](./memento.md)

### State

Permite modificar el comportamiento según el estado interno.

- [Ir a State](./state.md)

### Template Method

Define la estructura general de un algoritmo permitiendo especializar ciertos pasos.

- [Ir a Template Method](./template-method.md)

### Visitor

Permite agregar operaciones a estructuras existentes sin modificarlas.

- [Ir a Visitor](./visitor.md)

## Cuándo estudiar estos patrones

* Cuando aparecen grandes bloques de lógica condicional.
* Cuando múltiples objetos necesitan colaborar.
* Cuando existen workflows complejos.
* Cuando se implementan arquitecturas event-driven.

## Siguiente paso

Aplicar estos patrones en casos reales dentro de backend, frontend y arquitectura.
